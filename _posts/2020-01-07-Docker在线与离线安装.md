### docker安装的前提条件

linux内核版本需要高于3.10.可以通过uname -a命令进行确认

### docker在线安装

这个就没什么好记录的了， 看[官网教程](https://docs.docker.com/install/linux/docker-ce/ubuntu/)

以centos为例，其他系统请看官网教程

1. 卸载旧版本（没有安装过的请忽略)

```
sudo yum remove docker \
                  docker-client \
                  docker-client-latest \
                  docker-common \
                  docker-latest \
                  docker-latest-logrotate \
                  docker-logrotate \
                  docker-engine
```

2. 安装配置过程需要的依赖

```
sudo yum install -y yum-utils \
  device-mapper-persistent-data \
  lvm2
```

3. 添加docker官方维护的源仓库进服务器

```
sudo yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo
    
```

4. 安装docker环境

```
sudo yum install docker-ce docker-ce-cli containerd.io
```

- 进阶安装操作:安装特定版本的docker   

```
sudo yum install docker-ce-<VERSION_STRING> docker-ce-cli-<VERSION_STRING> containerd.io
```



### docker离线安装

什么情况会需要离线安装？

TO B行业中，如银行，电信之类的对内网安全要求高的都是不可以直接连接外网环境的仓库进行安装，并且又没有像互联网公司那样自己维护一个私有镜像仓库的时候就需要进行离线安装了。这个时候就到了考验各位同学对操作系统级别基础库如何构成的认知时候了，操作系统逃课的同学们这个时候就要注意了， 如何管理基层软件， 以前有yum, apt这种通过连接上一个已经搭建好的仓库源为我们管理各种的软件依赖， 一个install命令为你安装好所有软件，解决各种依赖了，但是一旦没有这个巨人的肩膀，就像瞬间从文明社会回退到了原始社会的感觉，重新开始学习钻木取火(卒)



#### 离线安装步骤

1. 先去[docker官方源仓库](https://download.docker.com/linux/)中进行下载安装包
2. 将下载好的包放在一个文件夹中
3. 进入这个文件夹中执行yum install * 或者 rpm -ivh *.rpm进行安装
4. 根据报错提示下载环境缺失的依赖包，然后重复2,3步骤，直至安装完成无报错(卒&*%$*#@#$%$&*%$@#)

### 离线时如何正确快速查找所需依赖包

1. 在官方的的yum源中进行离线访问仓库下载。特别是在发版时强依赖的包， 如docker这种，三个依赖包，互相依赖md5值的
2. 在操作系统发版官方仓库中进行查找   [centos仓库](http://mirror.centos.org/)
3. 在一些第三方仓库中查找，推荐一个[rpm finder](https://rpmfind.net/linux/RPM/index.html)
4. 最后一个终极大招，无招胜有招，直接google，但是只适用于知道软件包和对应的版本号的

### 如何根据包名确定自己需要的版本

举个常见的例子(ibm的那些土豪机器就不举例了)：

```
containers-common-0.1.37-1.el7.centos.x86_64.rpm

cockpit-composer-0.1.4-1.el7.noarch.rpm

docker-runc-1.0.0rc8+gitr3917_3e425f80a8c9-1.2.aarch64.rpm
```

上面的这些包的命令分别代表什么意思呢？

一般的rpm包命令规则为，括号表示非必填项:

```
包名+版本+(操作系统)+cpu的平台架构.rpm
X86_64 表示使用的是64位的x86指令集进行编译的

aarch64 标识使用的是64位的arm指令集进行编译的

noarch 表示没有平台架构限制,x86 和 arm都是用
```
